# Prompt para Cursor — Rediseño visual del detalle del lead v1.7.0

Bump `v1.6.0 → v1.7.0`. Cambio mayor de layout y sistema visual del detalle del lead. **No toca el dashboard** (eso queda para una iteración separada).

## Contexto

El detalle del lead v1.6.0 quedó funcionalmente bien, pero visualmente débil: todo el contenido del pane central (resumen, contexto, notas, actividades, adjuntos, chat IA) flota sobre el mismo fondo, sin separación visual entre secciones. El chat IA al pie compite por espacio vertical con el contenido informativo: cuando se expande, "come" la información del lead.

**La iteración corrige esto con dos cambios combinados**:

1. **Cambio de layout**: de 2 a 3 columnas. WhatsApp a la izquierda, información del lead al centro, chat IA en una columna dedicada a la derecha.
2. **Sistema visual**: cards con identidad propia, jerarquía clara, contraste sutil entre columnas. Sin multicolor.

**Va adjunta una imagen del mockup que validamos con Alejandro**. Tu trabajo es implementarlo lo más fiel posible respetando los detalles técnicos de este prompt.

## Decisiones cerradas

1. **Modal del detalle a 96vw × 96vh**, sin max-width, padding lateral mínimo. Corrige la desviación 2 de v1.6.0 (donde se conservaron `max-width: 1400px` / `max-height: 90vh`). Esto sí o sí — el layout de 3 columnas necesita el ancho.
2. **Tres columnas dentro del `.qida-detail-body`**:
   - **Pane WhatsApp** a la izquierda (~28%)
   - **Pane Información del lead** al centro (~40%)
   - **Pane Chat IA** a la derecha (~32%)
3. **El chat IA deja de estar al pie del pane central**. Pasa a tener su propia columna dedicada. En estado colapsado muestra header + 3 chips + input. En estado activo, los chips desaparecen y la conversación llena la columna verticalmente.
4. **Información del lead estructurada en cards independientes**: cada sección (Resumen IA, Contexto del cuidado, Notas internas, Próximas actividades, Adjuntos colapsable) en su propia card con borde sutil. Adjuntos sigue colapsable como en v1.6.0.
5. **Sin multicolor**: una sola paleta — blanco, grises sutiles, verde Qida para acentos. WhatsApp mantiene su crema típico solo en su columna porque las AFs lo esperan.
6. **Tipografía sin negritas heavy**: regular (400) para contenido, medium (500) solo para títulos y acentos. Nunca 600 o 700.

## Cambios concretos en `qida-widget.v1.js`

### 1. Modal a 96vw × 96vh cuando es detalle

Hoy `.qida-shell` está limitado a `max-width: 1400px` y `max-height: 90vh`. Hay que agregar un override cuando el modal está en vista detalle. Una forma limpia:

```javascript
// En syncShellHeader o en una nueva syncShellSizing()
var shell = document.getElementById('qida-shell');
if (!shell) return;
if (state.view === 'detail') {
    shell.classList.add('qida-view-detail');
} else {
    shell.classList.remove('qida-view-detail');
}
```

Y agregar al CSS:

```css
.qida-shell.qida-view-detail {
    max-width: none;
    max-height: none;
    width: 96vw;
    height: 96vh;
}
```

Esto NO afecta al dashboard, solo al detalle. Cuando se vuelve al dashboard con `back-to-dashboard`, la clase se quita y el shell vuelve a sus dimensiones originales.

### 2. Estructura HTML del detalle (3 columnas)

Reemplazar el contenido de `renderDetail()` para que use 3 paneles en lugar de 2:

```javascript
function renderDetail() {
    var lead = getLead(state.currentLeadId);
    if (!lead) return renderDashboard();
    return '<div class="qida-detail">'
        + '<div class="qida-detail-body">'
            + '<div class="qida-pane-wa">' + renderWhatsAppPane(lead) + '</div>'
            + '<div class="qida-pane-center">' + renderCenterPane(lead) + '</div>'
            + '<div class="qida-pane-ai">' + renderAiChat(lead) + '</div>'
        + '</div>'
    + '</div>';
}
```

Donde `renderCenterPane(lead)` agrupa el contenido informativo (Resumen IA, Contexto, Notas, Actividades, Followers si aplica, Adjuntos colapsable) sin incluir el chat IA al final.
